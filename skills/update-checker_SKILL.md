# アップデート自動チェッカー

## 目的

起動時に最新の更新情報を取得し、前回確認時点からの差分を日本語で提示する。
ただし**同日中に1回でも確認済みであれば、外部への通信は一切行わずスキップする**。

## 確認対象ソース（優先順）

| ソース | URL | 内容 |
|--------|-----|------|
| Code CHANGELOG | `https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md` | Claude Codeの詳細変更履歴 |
| API リリースノート | `https://docs.anthropic.com/en/release-notes/overview` | API・モデルの更新 |
| Apps リリースノート | `https://docs.anthropic.com/en/release-notes/claude-apps` | claude.ai・アプリの更新 |

---

## 実行手順

### ステップ0:【当日確認済みチェック】← 最初に必ず行う

`memory_user_edits` の `view` で `update_checker_last_run` の値を確認する。

**日付部分（YYYY-MM-DD）が今日と一致する場合 → 即座にスキップ**

以下の1行だけ出力して、ステップ1以降は実行せずユーザーの質問へ進む:

```
✅ 本日確認済み（[前回確認時刻]）— スキップ
```

**日付が今日と異なる、またはキーが存在しない場合 → ステップ1へ進む**

---

### ステップ1: 前回確認日時の取得

`update_checker_last_run` の値から前回日時を把握する。
存在しない場合は「初回実行」として扱い、直近7日分の更新を対象とする。

---

### ステップ2: ソースから更新情報を取得

以下の順で `web_fetch` を使って取得する:

```
1. https://raw.githubusercontent.com/anthropics/claude-code/main/CHANGELOG.md
   → text_content_token_limit: 4000

2. https://docs.anthropic.com/en/release-notes/overview
   → text_content_token_limit: 3000

3. https://docs.anthropic.com/en/release-notes/claude-apps
   → text_content_token_limit: 3000
```

取得に失敗したソースは `web_search` で補完する:
- クエリ例: `Claude Code changelog 2026 site:github.com`
- クエリ例: `Claude API release notes 2026 site:docs.anthropic.com`

---

### ステップ3: 差分の特定

前回確認日時以降に追加された項目のみを抽出する。
初回実行の場合は直近7日以内の項目を対象とする。

---

### ステップ4:【確認プレビューの提示】← 必須・スキップ禁止

差分をもとに、以下のフォーマットで簡潔なプレビューを出力する（詳細はまだ出さない）:

```
🔍 アップデート確認（前回: [日時] / 新着: [件数]件）

・Code: [バージョン] — [一行で変更の要点]
・API: [一行で変更の要点、なければ「変更なし」]
・Apps: [一行で変更の要点、なければ「変更なし」]
⚠️ 破壊的変更: [あれば件数と一言、なければ「なし」]

詳細を表示してメモリに記録しますか？
```

その後 `ask_user_input_v0` ツールで以下を表示する:

```json
{
  "questions": [
    {
      "question": "上記の更新内容の詳細を表示し、確認済みとして記録しますか？",
      "type": "single_select",
      "options": ["Yes（詳細を見る・記録する）", "No（今回はスキップ）"]
    }
  ]
}
```

---

### ステップ5:【回答に応じた分岐】

#### Yes を選んだ場合

以下のフォーマットで詳細レポートを出力する:

```
## 📋 最新アップデート情報

**確認日時**: [現在の日時]
**前回確認**: [前回日時 または「初回」]
**新着件数**: [件数]件

### 🆕 Code の更新
[バージョン番号と日付を示しながら要点を説明。技術用語は日本語に意訳し影響範囲も添える]

### 🔧 API / モデルの更新
[同様に要点を整理]

### 📱 アプリの更新
[同様に要点を整理]

### ⚠️ 注意が必要な破壊的変更
[該当なしの場合は「なし」と明記]

**次の一手**: [ユーザーが取るべき具体的なアクション、あれば]
```

出力後、`memory_user_edits` の `add` または `replace` で以下を記録する:

```
update_checker_last_run: [ISO 8601形式の現在日時（例: 2026-03-16T09:23:00+09:00）]
```

既存エントリがある場合は `replace` で上書き。行番号は `view` で確認してから指定すること。

#### No を選んだ場合

以下の1行だけ出力して終了。メモリへの記録は行わない（次回も同じ差分を確認できるよう保持する）:

```
スキップしました。次回起動時に再度確認します。
```

---

## 出力の注意事項

- 原文英語の変更内容は必ず日本語に意訳する（直訳ではなく、何が変わったかを伝える翻訳）
- バージョン番号・日付・固有名詞（Claude Code, MCP, OAuth等）はそのまま使用可
- 「バグ修正」「機能追加」「パフォーマンス改善」「廃止予定」の4カテゴリに分類できる場合は分類する
- 全体が20件超の場合は主要な上位10件に絞り「他 X 件の軽微な変更あり」と補足する
- `web_fetch` が失敗した場合は `web_search` にフォールバックし、失敗した旨を出力に添える

---

## 自動起動のルール

### トリガー1: 会話の最初のターン（最重要・無条件）

ユーザーが何を送ってきても、1ターン目は必ずこのスキルから始める。
ただし**ステップ0で当日確認済みと判定された場合は1行出力のみ**で即座にユーザーの質問へ進む。

冒頭に以下を一行追加する:
> 🔄 *起動時チェックを実行しました*

### トリガー2: キーワード検出（会話途中）

会話中に以下のいずれかが出たとき（ステップ0の当日スキップは適用しない）:
「最新情報」「アップデート」「更新」「changelog」「新機能」「何が変わった」
「最近の変更」「新しくなった」「バージョン」「リリースノート」「変更点」「リリース」

この場合はステップ0をスキップし、ステップ1から実行する。
